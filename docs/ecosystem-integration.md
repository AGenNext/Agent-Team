# Agent-Team Ecosystem Integration Guide

Agent-Team provides reusable goal-oriented agent teams.

It intentionally delegates specialized responsibilities to other repositories.

## Ownership Map

| Responsibility | Owning Repository |
|---|---|
| Objective contracts | Agent-Objective |
| Team blueprints | Agent-Blueprint |
| Policy constraints | Agent-Constraints |
| Skill definitions | Agent-Skills |
| Model/provider routing | Model-Router |
| Benchmarks | Agent-Bench |
| Evaluation and CLEAR scoring | Agent-Eval |
| Trust and provenance | Agent-Trust |
| Analytics and metrics | Agent-Analytics |
| Research intelligence | Agent-Research |
| Runtime adapters | Agent-Frameworks |
| Environment contracts | Agent-Environment |
| Artifact schemas | Agent-Graph |
| Product implementation | Agent-Knowledge |

## Core Principle

```text
Agent-Team coordinates work.
Specialized repositories own their domains.
```
